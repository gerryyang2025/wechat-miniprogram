## 2. Nano Banana 出图统一规则

### 2.1 输出要求

- 每次只生成 `1` 张页面图
- 输出为 `微信小程序移动端单页交互图`
- 中文界面
- 竖屏比例
- 不要设备外框
- 不要手持场景
- 不要多页面拼贴
- 不要设计说明浮层
- 不要英文占位符
- 不要低保真线框图

### 2.2 微信小程序画布参数

统一建议：

- 画面比例：`9:19.5`
- 逻辑稿尺寸：`390 x 844`
- 如工具支持像素尺寸，导出尺寸建议：`1170 x 2532` 或 `1242 x 2688`
- 页面可视宽度按 `微信小程序 750rpx` 设计体系理解
- 顶部安全区预留：`88 ~ 120px` 逻辑高度
- 底部安全区预留：
  - 含底部 Tab 的页面：`100 ~ 120px`
  - 无底部 Tab 的页面：`40 ~ 60px`
- 页面左右安全边距：`16 ~ 24px`
- 卡片圆角：`16 ~ 20px`
- 卡片间距：`12 ~ 16px`
- 模块间距：`20 ~ 28px`

### 2.3 移动终端显示约束

为保证在微信小程序和移动终端预览时信息清晰，建议遵循：

- 单张页面图首屏必须能看清页面标题、主模块和主 CTA（Call To Action）
- 正文字号视觉上不低于 `14px`
- 辅助信息字号视觉上不低于 `12px`
- 标题字号建议 `20 ~ 24px`
- 单行不要塞过长文本，避免图片缩放后不可读
- 卡片列表尽量控制在 `2 ~ 4` 组核心内容，避免过密
- 重要按钮必须出现在用户拇指可达区域或首屏下半区
- 底部 Tab、固定操作栏、播放器控制区不要被裁切
- 不要生成过长页面截图，优先输出 `一屏或一屏半` 的页面交互图
- 导出图需适合在手机端查看，避免太淡的灰色文字和过细分隔线

### 2.4 视觉风格要求

- 风格：`中高保真产品交互图`
- 平台感：明显符合 `微信小程序`
- 当前风格基准：优先参考 `prototype-images/original/demo1.jpg`
- 气质：专业、明亮、可信，带轻科技感和内容学习平台感
- 页面整体结构：顶部大面积蓝紫渐变氛围区，主体内容区以白色或极浅冷灰色为主
- 背景主色：蓝紫、靛蓝、亮蓝、少量青蓝高光
- 文字：深灰色或白色，按背景明暗切换
- 强调色：蓝紫色、亮蓝色、少量青色高光
- 状态色：绿色只用于 `直播中 / 已完成` 等状态
- 卡片：白色或浅蓝白色，圆角 `16px` 左右，轻阴影
- 图标：简洁线性图标
- 页面留白充足，信息层级清楚
- Banner 美术风格：蓝紫科技学习风、2.5D 插画或轻拟物内容模块、大标题高对比中文字效

### 2.5 统一示例数据

为保证多张页面图风格一致，建议统一使用以下示例内容：

- 店铺名：`时昕有点懒`
- 品牌副标题：`个人IP与私域运营内容平台`
- 讲师名：`Gerry`
- 课程 A：`个人IP内容变现实战课`
- 课程 B：`AI私域运营入门课`
- 课程 C：`高转化训练营设计方法课`
- 训练营：`7天私域增长训练营`
- 直播：`今晚20:00 私域运营直播答疑`
- 会员：`年度会员计划`

### 2.6 提示词写法建议

每个页面建议按以下结构描述：

1. 页面类型
2. 页面目标
3. 必备模块
4. 关键状态
5. 示例文案
6. 输出约束

### 2.7 当前推荐出图优先级

为方便原型阶段快速推进，当前建议按下面顺序使用 `Nano Banana AI` 生成页面提示词。

对应保存目录与文件命名规则见 [prototype-images/nano-banana/README.md](./prototype-images/nano-banana/README.md)。

第一优先级：先跑通用户主路径

1. 首页
2. 商品详情页
3. 课程播放页
4. 学习页
5. 训练营详情页
6. 直播详情页
7. 直播间页
8. 我的页

第二优先级：补齐基础服务页

9. 消息通知页
10. 咨询反馈页
11. 系统设置页

第三优先级：补齐商家工作台主路径

12. 商家工作台首页
13. 商品管理页
14. 直播管理页
15. 用户管理页
16. 内容运营页

建议说明：

- 前 `8` 个页面可以最快形成 `首页 -> 商品详情 -> 课程播放 / 训练营详情 / 直播详情 -> 直播间 -> 学习 / 我的` 的完整用户体验链路
- 第 `9 ~ 11` 个页面用于补齐基础服务能力，适合在主路径稳定后集中出图
- 第 `12 ~ 16` 个页面用于补齐商家侧原型，便于后续同步推进工作台编码实现

- 搜索
- Banner

- 课程列表首屏露出 `2` 张课程卡片

1. 首页
2. 商品详情页
3. 课程播放页
4. 学习页
5. 训练营详情页
6. 直播详情页
7. 直播间页
8. 我的页
9. 消息通知页
10. 咨询反馈页
11. 系统设置页
12. 商家工作台首页
13. 商品管理页
14. 直播管理页
15. 用户管理页
16. 内容运营页

### 5.2 生成后的检查项

- 是否仍然符合微信小程序风格
- 是否保持统一品牌和示例数据
- 是否能够清楚表达主路径跳转关系
- 是否能看出该页的核心 CTA（Call To Action）
- 是否适合作为前端页面拆分和编码参考

### 5.3 二次修订建议

如果首轮生成效果不稳定，优先补充以下信息重新生成：

- 当前页唯一目标
- 需要展示的卡片数量
- 哪个模块最重要
- 哪个按钮必须显眼
- 当前页面是 `直播中 / 未开播 / 已结束` 还是 `学习中 / 已完成`
