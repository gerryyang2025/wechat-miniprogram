## 2. 页面设计原则

结合 `../prototype-images/original/` 目录下的 demo 页面图，一期用户端页面结构遵循以下原则：

- 一级结构尽量简单，优先使用 `首页 / 学习 / 我的` 三个主入口
- 首页承担 `发现、推荐、进入`
- 学习页承担 `已购内容管理、学习数据、快速进入学习`
- 我的页承担 `账户、权益与基础服务入口`
- 商品详情页承担 `信任建立、内容预览、咨询与进入`
- 播放页承担 `学习、资料查看、咨询`
- 当前一期 `MVP` 不纳入平台审核端与付费链路页面
- 当前一期 `MVP` 不纳入 demo 图中的独立练习、考试、作业本、付费问答、我的圈子、我的拼团、我的证书、兑换码、活动参与记录、推广中心等非核心页面
- 当前一期优先完成核心需求对应的产品原型页，视频播放、直播观看、学习进度等功能按模块逐步接入

## 4. 用户端页面清单

### 4.1 首页

页面目标：

- 帮助用户快速发现课程
- 承接营销内容和主推活动
- 提供从浏览到进入内容的最短路径

核心模块：

- 顶部搜索框
- Banner 轮播
- 已购课程区
- 课程推荐列表
- 可扩展的训练营 / 直播 / 会员推荐位

主要来源参考：

- `demo1.jpg`

### 4.2 学习页

页面目标：

- 帮助用户集中管理已购买内容
- 提供学习数据反馈与快速继续学习路径

核心模块：

- 学习数据看板
  - 今日学习时长
  - 累计学习时长
  - 累计学习天数
- 已购课程搜索
- 已购课程列表
- 空状态页

主要来源参考：

- `demo2.jpg`

### 4.3 我的页

页面目标：

- 承接账户、服务与权益入口
- 提供用户侧自助管理入口

核心模块：

- 用户头像、昵称、手机号绑定
- 会员入口
- 学习服务入口
- 服务功能列表

主要来源参考：

- `demo3.jpg`
- `demo4.jpg`

### 4.4 商品详情页

页面目标：

- 帮助用户理解内容价值并进入学习或直播
- 建立商品信任感
- 让用户在购买前理解内容结构和权益

核心模块：

- 顶部封面或预览区
- 商品信息区
- 商品标签与节数信息
- 权益说明
- 消费者保障 / 商家资质
- 简介与目录切换
- 底部咨询和进入按钮

主要来源参考：

- `demo6.jpg`

### 4.5 课程播放页

页面目标：

- 支撑课程消费过程
- 降低用户对资料、目录和咨询的切换成本

核心模块：

- 视频 / 音频 / 图文播放器
- 标题与时长信息
- 学习资料 / 附件查看
- 消费者保障入口
- 咨询入口

主要来源参考：

- `demo5.jpg`

### 4.6 训练营详情页

页面目标：

- 展示训练营计划
- 串联打卡、任务与公告

核心模块：

- 训练营介绍
- 任务 / 课表
- 打卡入口
- 公告区

主要来源参考：

- 结合 [DESIGN.md](./DESIGN.md) 中的训练营能力架构

### 4.7 直播详情页与直播间页

页面目标：

- 展示直播基础信息和观看条件
- 承接直播进入、观看和后续回放

核心模块：

- 直播封面与主题信息
- 开播时间、讲师信息、准入说明
- 进入直播间按钮
- 直播画面区与互动区
- 准入失败、未开播、已结束等占位状态

主要来源参考：

- 结合 [CORE_REQUIREMENTS.md](./CORE_REQUIREMENTS.md) 中的直播授课需求
- 结合 [MVP_FEATURES.md](./MVP_FEATURES.md) 中的 `直播详情页 / 直播间页` 原型优先级

## 7. 页面跳转关系

### 7.1 用户端核心跳转

首页：

- 搜索 -> 搜索结果页或商品列表页
- Banner -> 商品详情页 / 活动页
- 已购课程区 -> 学习页 / 课程播放页 / 训练营详情页
- 推荐课程卡片 -> 商品详情页

学习页：

- 已购课程卡片 -> 课程播放页 / 训练营详情页
- 搜索结果 -> 课程播放页 / 训练营详情页

我的页：

- 会员入口 -> 会员详情页
- 学习记录 -> 学习页
- 观看中的直播 / 训练营 -> 直播详情页 / 训练营详情页
- 咨询反馈 -> 咨询反馈页
- 系统设置 -> 系统设置页

商品详情页：

- 课程目录 -> 章节详情或课程播放页
- 咨询 -> 咨询页 / 客服会话
- 进入内容 -> 课程播放页 / 训练营详情页 / 直播详情页

直播详情页：

- 进入直播间 -> 直播间页
- 回放入口 -> 课程播放页或回放播放页

课程播放页：

- 附件 -> 附件预览 / 下载页
- 咨询 -> 咨询页 / 客服会话

### 7.2 商家端核心跳转

- 工作台快捷入口 -> 商品 / 直播 / 用户 / 内容运营页
- 商品列表 -> 编辑商品页
- 直播列表 -> 直播详情页
- 用户列表 -> 用户详情页
- 用户详情 -> 学习档案 / 跟进记录
- 内容运营 -> Banner / 推荐位配置页

## 9. 原型阶段建议

在进入原型前，建议按下面顺序出图：

1. 首页
2. 学习页
3. 我的页
4. 商品详情页
5. 课程播放页
6. 训练营详情页
7. 直播详情页
8. 直播间页
9. 商家工作台首页
10. 商品管理页

这 10 个页面可以最快形成一套可讨论、可评审的主流程原型。

### 9.1 原型阶段实现建议

为提升 MVP 推进速度，建议原型阶段遵循以下实现方式：

- 页面先完成信息布局、模块占位和关键 CTA（Call To Action）定义
- 页面联调阶段优先使用静态数据或 Mock 数据，不依赖全部后端接口完成
- `课程播放页` 第一阶段先完成播放器区域、目录区、资料区和进度区的页面结构
- `直播详情页 / 直播间页` 第一阶段先完成直播信息、准入说明、互动区和占位态页面结构
- 视频播放、直播观看、学习进度、训练营打卡等能力在原型稳定后再按模块接入
