## 1. 文档目的

这份文档用于记录 `private-domain-operation` 当前已经实现的功能、待实现的功能，以及下一步建议的修改顺序。

与其他文档的分工如下：

- [DESIGN.md](./DESIGN.md)：记录产品定位、能力架构和平台策略
- [MVP_FEATURES.md](./MVP_FEATURES.md)：记录一期功能范围、优先级和验收标准
- [IA_PAGES.md](./IA_PAGES.md)：记录页面结构和页面职责
- `IMPLEMENTATION_TRACKER.md`：只记录当前代码实现进度与下一步计划

## 2. 当前状态说明

状态定义：

- `已完成`：已经落地到代码，可在小程序中直接看到
- `部分完成`：页面或结构已经有了，但仍是静态占位或跳转未接通
- `未开始`：文档中已有规划，但当前代码还没实现
- `后期实现`：明确不进入当前阶段

## 8. 下一步建议修改计划

### 8.1 第一优先级：补齐用户主路径

建议顺序：

1. 直播回放状态细化
2. 我的页服务入口
3. 商家工作台首页
4. 商品管理页
5. 直播管理页

目标：

- 跑通 `首页 -> 商品详情 -> 课程播放 / 训练营详情 / 直播详情 -> 直播间`
- 让课程、训练营、直播这三类内容的“查看详情 -> 进入交付”更完整
- 把剩余高频入口从占位提示逐步变成真实页面或真实状态

### 8.2 第二优先级：整理本地数据与页面跳转

- 已新增统一的 `mock/` 数据目录
- 继续把首页 / 学习页 / 我的页中的静态数据抽离出去
- 统一页面之间的参数传递方式，如 `courseId`、`liveId`、`campId`
- 定义统一的课程视频 Mock 数据结构，如 `video_url`、`cover_url`、`duration_seconds`
- 先准备 `3 ~ 5` 个课程播放示例视频，优先上传到 `COS（Cloud Object Storage）`
- 为每个示例视频准备对应封面图与标题文案
- 课程播放页补充“资源准备中 / 播放失败 / 封面缺失”三类最小状态

### 8.3 第三优先级：补齐商家端核心原型

建议顺序：

1. 商家工作台首页
2. 商品管理页
3. 直播管理页
4. 用户管理页
5. 内容运营页

### 8.4 第四优先级：补齐基础服务页

- 消息通知页
- 咨询反馈页
- 系统设置页

### 8.5 第五优先级：工程整理

- 清理项目内 `.DS_Store`
- 统一全局颜色、圆角、阴影等样式变量
- 评估将 `prototype-images` 移出小程序项目目录，减少资源检查干扰

## 9. 当前建议结论

当前项目已经完成了 `首页 / 学习 / 我的` 三个一级入口的前端原型骨架，适合继续进入第二阶段开发。

最值得立刻开始的工作不是接后端，而是：

1. 补齐用户主路径页面
2. 打通页面间真实跳转
3. 抽离统一的本地 Mock 数据

只要这三步完成，`private-domain-operation` 的 MVP 原型阶段就会从“可浏览首页”提升到“可完整走查主要业务路径”。
