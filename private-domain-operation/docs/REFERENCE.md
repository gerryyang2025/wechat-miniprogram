## 1. 文档目标

本文档用于补充 `private-domain-operation` 在实现前的 GitHub 开源项目参考，帮助团队更快完成以下工作：

- 判断当前产品与哪些成熟开源项目最相似
- 确定应该优先借鉴的产品结构、页面结构与业务模型
- 避免从零设计课程、直播、分销、订单、后台工作台等通用模块
- 明确“哪些项目适合借鉴产品能力，哪些项目适合借鉴微信小程序实现方式”

注意：

- 本文档的排名是基于 `产品相似度` 和 `MVP 落地参考价值` 的综合判断，不是单纯按 GitHub star 排名
- 当前目标技术栈为 `微信小程序 + Golang + Gin + MySQL + etcd + COS`，以下项目大多不能直接作为技术底座复用，更适合作为 `产品能力参考`、`页面结构参考`、`领域模型参考`

## 2. 评估维度

本次相似度排序主要参考以下维度：

1. `知识付费 / 在线教育` 能力贴合度
2. `微信生态 / 小程序 / 私域运营` 贴合度
3. `直播 / 分销 / 营销 / 交易` 贴合度
4. `后台工作台 / 用户运营 / 数据分析` 贴合度
5. 对当前 `MVP` 的可借鉴价值

综合相似度为主观工程评估分值，仅用于本项目实现准备。

## 4. 项目逐项分析

### 4.1 `edusoho/edusoho`

仓库地址：

- <https://github.com/edusoho/edusoho>

当前公开信息摘要：

- GitHub 页面显示约 `1.9k stars / 735 forks`
- README 明确覆盖 `直播录播教学`、`自动化考试系统`、`任务式教学计划`
- 同时包含 `微信营销平台`、`分销 / 砍价 / 拼团`
- 还强调 `网校小程序` 与微信生态联动

为什么排第 1：

- 它是这批项目里最接近“知识付费 + 在线教学 + 营销招生 + 企业培训”的综合产品
- 既有课程与教学计划，也有营销与数据管理
- 对你现在的 `课程 + 训练营 + 教测练 + 分销 + 小程序` 目标最接近

建议重点借鉴：

- 课程、训练营、直播、考试这几类产品能力如何共存
- 学员、讲师、班主任、管理员的角色边界
- 网校后台如何同时承接 `教学管理 + 订单管理 + 营销管理`
- 企业培训和知识付费共用一套底座的方式

不适合直接照搬的部分：

- 体系偏大，远超一期 `MVP`
- 技术栈不是当前项目目标技术栈
- 开源产品形态更偏完整平台，不适合直接按全部功能照抄

### 4.2 `Qsnh/meedu`

仓库地址：

- <https://github.com/Qsnh/meedu>

当前公开信息摘要：

- GitHub 页面显示约 `3.8k stars / 1k forks`
- README 明确定位为 `在线网校 / 知识付费 / 线上培训解决方案`
- 开源版本支持 `线上点播、课程购买、网校装修、手机号登录注册、学习统计、角色管理`
- 商业版说明中进一步覆盖 `直播课、考试练习、电子书、图文、兑换码、微信小程序`

为什么排第 2：

- 它比 EduSoho 更轻，更像一个可快速落地的知识付费 MVP
- 很适合作为 `课程上架 -> 支付 -> 学习 -> 统计` 这条主链路的参考
- 对“个人讲师 / 中小机构”这类目标用户更贴近

建议重点借鉴：

- 课程商品、学习页、统计页的最小闭环
- 知识付费产品如何拆成 `管理后台 + API + H5/PC`
- 用户注册、课程购买、学习统计的简化实现方式
- 适合用来压缩你们自己的一期范围，避免做得过重

不适合直接照搬的部分：

- 开源版与商业版能力边界明显
- 你当前特别重视的 `分销 / 直播带货 / 小程序` 在开源版里不够完整

### 4.3 `crmeb/CRMEB`

仓库地址：

- <https://github.com/crmeb/CRMEB>

当前公开信息摘要：

- GitHub 页面显示约 `8.5k stars / 1.9k forks`
- README 强调支持 `小程序商城、H5、公众号、PC、App`
- 功能覆盖 `分销、拼团、砍价、秒杀、优惠券、会员等级、小程序直播、页面 DIY`

为什么排第 3：

- 它不是教育项目，但它非常接近你们产品里“私域运营管理工具”的另一半
- 如果把 `课程 / 训练营 / 会员` 看成特殊商品，CRMEB 在交易、营销、直播带货、会员、推广这层很有借鉴价值

建议重点借鉴：

- 小程序商城的交易链路
- 分销、优惠券、直播带货、会员体系
- 商家后台的商品、订单、营销、页面装修模块
- 微信生态里的商城与用户运营方式

不适合直接照搬的部分：

- 教学、学习进度、作业、训练营这些教育能力基本需要你们自己补
- 代码和模块会明显偏电商，不适合作为知识付费产品的唯一参考

### 4.4 `lilishop/lilishop`

仓库地址：

- <https://github.com/lilishop/lilishop>

当前公开信息摘要：

- GitHub 页面显示约 `4.2k stars / 1k forks`
- README 强调覆盖 `PC、H5、小程序、APP`
- 重点能力包括 `商家入驻`、`分布式架构`、`完整电商业务模块`

为什么排第 4：

- 它非常适合作为“商家工作台与后台模块划分”的参考
- 对商品、订单、促销、店铺、运营、统计这些后台模块的结构借鉴价值高

建议重点借鉴：

- 商家工作台信息架构
- 商品、订单、店铺、营销、统计的后台拆分方式
- 多端商城如何共享一套业务底座

不适合直接照搬的部分：

- 偏平台电商与多商户，复杂度比你们一期要高
- 教育内容交付和学习体系基本缺失

### 4.5 `mindskip/xzs`

仓库地址：

- <https://github.com/mindskip/xzs>

当前公开信息摘要：

- GitHub 页面显示约 `3.9k stars / 759 forks`
- README 明确支持 `web 端和微信小程序`
- 功能覆盖 `任务中心、考试、固定试卷、时段试卷、错题本、消息中心`
- 同时有 `管理系统功能` 和 `小程序功能`

为什么排第 5：

- 它不是知识付费商城，但对你们的 `训练营 / 练习 / 考试 / 消息中心` 很有参考价值
- 如果后续 `作业本 / 练习中心 / 考试中心` 进入一期增强或二期，它会比纯商城项目更有帮助

建议重点借鉴：

- 教测练体系的最小数据模型
- 学员端与管理端在考试场景下的页面拆分
- 小程序端任务、消息、考试记录的交互方式

不适合直接照搬的部分：

- 不覆盖交易、课程商品、分销、直播带货
- 更偏考试系统，而不是私域知识付费经营平台

### 4.6 `thundersword/fire-shop-lite`

仓库地址：

- <https://github.com/thundersword/fire-shop-lite>

当前公开信息摘要：

- GitHub 页面显示约 `446 stars / 156 forks`
- README 写明是 `电商小程序前端开源项目`
- 功能包括 `拼团、砍价、分销、商品海报生成、优惠券`
- 说明中提到 `直播插件`
- README 同时明确写了 `已停止维护`

为什么排第 6：

- 它的优势不在业务完整度，而在于“小程序前端页面和插件接入方式”比较直接
- 如果你们要快速落微信小程序的商城页、海报分享、直播插件接入，它是很直观的前端参考

建议重点借鉴：

- 小程序商城前端页面组织方式
- 分销与海报分享相关前端交互
- 直播插件接入示例

不适合直接照搬的部分：

- 已停止维护
- 只有前端参考价值，不适合作为完整业务底座

### 4.7 `tumobi/nideshop-mini-program`

仓库地址：

- <https://github.com/tumobi/nideshop-mini-program>

当前公开信息摘要：

- GitHub 页面显示约 `8.3k stars / 2.3k forks`

- 功能包括 `首页、分类、商品详情、搜索、专题、完整购物流程、会员中心`
- 同时注明 `当前版本功能还未完善，请勿商用`

为什么排第 7：

- 但它和你们当前目标的教育、训练营、直播、分销体系差距较大

建议重点借鉴：

- 微信小程序商城基础信息架构
- 商品详情、购物车、订单中心的页面拆法

不适合直接照搬的部分：

- 项目本身不完整
- 不包含知识付费、学习进度、训练营、教测练、分销结算等关键能力

## 5. 推荐使用策略

不建议只盯一个项目看。更适合当前 `MVP` 的方式是 `组合借鉴`：

### 5.1 产品结构主参考

- `edusoho/edusoho`
- `Qsnh/meedu`

用途：

- 用于确定课程、训练营、直播、会员、用户运营、数据分析之间的主结构

### 5.2 私域交易与营销主参考

- `crmeb/CRMEB`
- `lilishop/lilishop`

用途：

- 用于确定分销、优惠券、直播带货、商家后台、订单与营销链路

### 5.3 教测练补充参考

- `mindskip/xzs`

用途：

- 用于补强练习、考试、任务、消息中心、学习记录

### 5.4 小程序前端交互补充参考

- `thundersword/fire-shop-lite`
- `tumobi/nideshop-mini-program`

用途：

- 用于借鉴小程序商城页、会员页、购物流程和直播插件接入方式

## 6. 对当前项目的直接建议

如果目标是尽快开始实现 `MVP`，建议按下面顺序吸收这些参考项目：

1. 先参考 `MeEdu + EduSoho`
原因：先把知识付费、课程、训练营、学习链路定稳。

2. 再参考 `CRMEB + Lilishop`
原因：再把分销、支付、直播带货、商家工作台和营销链路定稳。

3. 最后参考 `XZS + fire-shop-lite + NideShop`
原因：再补齐教测练和微信小程序前端实现细节。

## 7. 当前结论

截至当前调研，没有发现一个 GitHub 开源项目能够同时完整满足以下四点：

- 微信小程序优先
- 知识付费与训练营
- 私域运营与分销
- 直播带货与提现

因此，对 `private-domain-operation` 来说，最合理的实现策略不是“选一个仓库直接二开”，而是：

- 以你们自己的 `产品设计文档 + MVP 功能清单` 为主
- 以 `EduSoho / MeEdu` 作为教育产品参考
- 以 `CRMEB / Lilishop` 作为私域交易与营销参考
- 以 `XZS / fire-shop-lite / NideShop` 作为能力补充和前端交互参考
