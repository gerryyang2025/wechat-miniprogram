# private-domain-operation 一期可联调 MVP 设计

日期：`2026-06-25`

状态：`已确认设计，待实现计划`

## 1. 背景

`private-domain-operation` 当前已经具备较完整的微信小程序原型、前端 API fallback 层、Gin 后端路由和 SQLite migration 基础。下一阶段目标不是继续扩展静态页面，而是打通一条真实可联调的纵向闭环，证明前端、后端、SQLite、课程媒资和学习进度可以协同工作。

## 2. 目标

第一轮一期可联调 MVP 打通以下链路：

```text
真实 wx.login
  -> openid 白名单识别商家
  -> SQLite 读取 / 保存课程数据
  -> 商家编辑 1 门种子课程
  -> 学员端看到课程更新
  -> 学员进入课程播放
  -> 学习进度上报到 SQLite
  -> 商家侧看到最小学情摘要
```

核心目标：

- 证明真实小程序登录、后端鉴权和商家角色识别可用
- 证明 SQLite 可作为当前一期主业务数据库
- 证明商家编辑内容后，学员端能读取到更新结果
- 证明学习进度可以写入并再次读取
- 保留非核心页面的 mock fallback，降低第一轮联调风险

## 3. 非目标

第一轮不做以下能力：

- 从零创建课程
- 删除课程、复制课程、批量编辑、课节拖拽排序
- 小程序端视频上传或 SFTP 上传 UI
- 训练营真实打卡、公告发布
- 直播准入、直播互动、微信直播组件接入
- 会员购买、权益开通、订单、支付、退款
- 分销、佣金、提现、平台审核端
- 全量 mock 数据迁移到 SQLite

## 4. 范围

本轮只覆盖最小双端闭环：

- `1` 个商家
- `1` 门种子课程
- `2-3` 个课节
- 每个可播放课节维护标题、HTTPS 视频地址和时长
- 资源未准备好的课节可保留空视频地址，并以 `resourceState=preparing` 呈现
- 课程维护标题、简介、上架状态和 HTTPS 封面地址
- 首页推荐位展示该课程
- 学员学习进度记录最近课节、完成课节数、进度百分比和最近学习时间
- 商家侧展示学习人数、完成人数、平均进度和最近学习时间

训练营、直播、会员、通知、设置、咨询和内容运营页面继续保留现有 mock fallback。

## 5. 方案选择

采用方案：`纵向最小闭环`

选择原因：

- 它最能证明系统从商家侧写入到学员侧消费再到商家侧看数据的真实闭环
- 它避免把第一轮时间消耗在全量 mock 数据迁移上
- 它比单纯后端底座更容易验收，也比全量 P0 SQLite 化更可控

未选方案：

- `后端底座先行`：后端结构更扎实，但短期缺少可见的双端产品闭环
- `P0 只读接口全面 SQLite 化`：覆盖面更广，但会被训练营、直播、会员等数据迁移拖慢

## 6. 模块设计

### 6.1 认证模块

小程序调用 `wx.login` 获取 code，后端调用微信 `code2session` 换取 openid。

后端行为：

- 使用 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET` 调用微信接口
- 成功后按 openid upsert 用户
- 如果 openid 命中 `MERCHANT_OPENIDS` 白名单，用户角色包含 `merchant`
- 签发业务 token，前端保存到本地存储

第一轮商家身份只通过 openid 白名单识别，不建设商家邀请或成员管理后台。

### 6.2 SQLite 数据模块

后端使用 SQLite 作为主业务数据库。

要求：

- 默认数据库路径为 `data/pdo.db`
- 开启 `WAL`
- 开启 `foreign_keys`
- migration 继续放在 `backend/migrations`
- seed 数据通过独立命令或 `optools seed` 写入
- 数据库文件不提交仓库

最小 seed 数据：

- 商家
- 命中白名单的商家用户
- 普通学员用户
- 1 门课程商品
- 2-3 个课节
- 对应媒资记录
- 首页推荐位
- 1 条初始学习进度

### 6.3 课程读取模块

学员端核心课程链路优先读取真实后端：

- 首页推荐课程
- 商品列表
- 商品详情
- 课程播放页
- 学习页课程进度摘要

课程播放页只消费后端返回的标准字段：

- `videoUrl`
- `coverUrl`
- `resourceState`
- `chapters`
- `progressSummary`

### 6.4 商家课程编辑模块

新增轻量课程编辑页，从商品管理页点击“编辑”进入。

字段范围：

- 课程标题
- 课程简介
- 上架状态
- 封面 HTTPS 地址
- 课节标题
- 课节 HTTPS 视频地址
- 课节时长

保存后写入 SQLite。学员端再次进入首页、商品详情或课程播放页时，应看到商家保存后的数据。

第一轮只编辑已有种子课程，不支持新建课程。

### 6.5 学习进度与学情模块

学员端上报：

- 当前课节
- 是否完成
- 最近播放位置秒数

后端保存并计算：

- 已完成课节数
- 总课节数
- 进度百分比
- 最近学习时间
- 最近学习位置

商家侧展示课程最小学情摘要：

- 学习人数
- 完成人数
- 平均进度
- 最近学习时间

## 7. 接口设计

### 7.1 真实化现有接口

- `POST /api/v1/auth/wechat-login`
  - 从 mock token 改为真实 `wx.login code -> code2session -> token`

- `GET /api/v1/home`
  - 首页推荐课程和已购课程摘要读取 SQLite 中的种子课程

- `GET /api/v1/products`
  - 返回 SQLite 中已上架的种子课程商品

- `GET /api/v1/products/{product_id}`
  - 返回课程详情、简介、目录和播放入口

- `GET /api/v1/courses/{course_id}/player`
  - 返回课节、媒资、播放地址和进度摘要

- `GET /api/v1/learning`
  - 学习页课程列表读取当前用户学习进度

- `GET /api/v1/learning/courses/{course_id}/progress`
  - 查询指定课程进度

- `POST /api/v1/learning/courses/{course_id}/progress`
  - 上报最近学习位置和完成状态

- `GET /api/v1/merchant/products`
  - 商品管理页读取种子课程

### 7.2 新增商家接口

- `GET /api/v1/merchant/courses/{course_id}/edit`
  - 获取课程编辑页数据

- `PUT /api/v1/merchant/courses/{course_id}`
  - 保存课程标题、简介、状态、封面、课节和视频地址

- `GET /api/v1/merchant/courses/{course_id}/analytics`
  - 返回最小学情摘要

### 7.3 继续 fallback 的接口

以下接口本轮不要求真实落库：

- 训练营相关接口
- 直播相关接口
- 会员权益接口
- 消息通知接口
- 系统设置接口
- 咨询反馈接口
- 内容运营接口

## 8. 前端设计

### 8.1 API 配置

`app.js` 支持配置本地或测试后端 `baseUrl`。未配置 `baseUrl` 时继续使用 mock fallback。

### 8.2 登录

前端通过 `wx.login` 调后端登录接口。成功后保存 token。商家白名单用户可访问商家接口，普通用户访问商家接口应得到权限错误。

### 8.3 商品管理页

商品管理页的“编辑”按钮从 toast 改为跳转课程编辑页。

### 8.4 新增课程编辑页

页面内容：

- 课程基础信息表单
- 课节编辑列表
- 保存按钮
- 保存成功反馈

保存成功后可以返回商品管理页，也可以留在当前页继续编辑。

### 8.5 学员端核心页面

首页、商品列表、商品详情、课程播放页和学习页继续复用现有 API 服务层。配置 `baseUrl` 后优先请求后端，接口失败且允许 fallback 时仍可展示 mock 数据。

## 9. 后端设计

建议后端分层：

- `internal/config`
  - 读取 SQLite 路径、微信配置、商家 openid 白名单

- `internal/db`
  - 打开 SQLite
  - 设置 WAL 和 foreign_keys
  - 执行 migration

- `internal/repository`
  - 用户、课程、课节、媒资、学习进度、商家查询封装

- `internal/service`
  - 登录、课程编辑、学习进度、学情聚合

- `internal/handler`
  - 参数解析、鉴权、统一响应

当前 handler 中的硬编码数据应逐步迁移到 repository/service，第一轮只迁移核心课程闭环需要的数据。

## 10. 校验与错误处理

登录：

- 微信 `code2session` 失败返回登录失败
- 缺少微信配置时返回明确配置错误

权限：

- 非登录用户访问需要鉴权的接口返回 `401`
- 非商家用户访问商家接口返回 `403`

课程：

- 课程不存在返回 `404`
- 保存课程时至少校验标题、状态、课节标题
- 视频地址和封面地址第一轮要求为 `https://`，空值只允许用于未准备好的课节

学习进度：

- 课程或课节不存在返回 `404`
- 上报进度秒数小于 0 时按参数错误处理
- 完成课节后进度百分比应重新计算

## 11. 验收标准

### 11.1 真实登录

- 小程序调用 `wx.login`
- 后端成功换取 openid
- 后端签发 token
- 命中白名单 openid 的用户具备商家角色
- 普通用户不能访问商家编辑接口

### 11.2 商家编辑课程

- 商家进入商品管理页
- 点击种子课程“编辑”
- 修改课程标题、简介、状态、封面、课节标题、视频地址、时长
- 保存成功
- 刷新后数据仍存在 SQLite

### 11.3 学员看到更新

- 学员进入首页、商品列表、商品详情和课程播放页
- 能看到商家保存后的课程标题、简介、课节和视频地址

### 11.4 课程播放与进度

- 学员进入课程播放页
- 切换课节或标记完成后，进度写入 SQLite
- 重新进入课程后，最近学习位置、完成课节数和进度百分比保持一致

### 11.5 商家学情摘要

- 商家侧能看到课程学习人数
- 商家侧能看到完成人数
- 商家侧能看到平均进度
- 商家侧能看到最近学习时间

### 11.6 fallback 不阻塞

- 训练营、直播、会员、通知、设置、咨询仍可依靠 mock fallback 正常展示
- 核心课程链路在配置 `baseUrl` 后优先请求后端
- 未配置 `baseUrl` 时小程序仍能按现有 mock 方式运行

### 11.7 工程验证

- 后端 `go test ./...` 通过
- migration 能在空 SQLite 数据库执行
- seed 命令可重复执行或给出明确已存在处理
- 核心接口返回结构与现有前端 API 服务层兼容

## 12. 实施顺序建议

1. SQLite 连接、migration 执行、seed 数据
2. 真实微信登录与 openid 白名单商家识别
3. 核心课程读取接口落库
4. 学习进度接口落库
5. 商家课程编辑接口
6. 商家课程编辑页
7. 商家学情摘要
8. 前端 baseUrl 联调与 fallback 验证

## 13. 风险与处理

- 微信登录依赖外网与 AppSecret：开发环境需要明确配置方式；配置缺失时应给出清晰错误。
- 小程序合法域名限制：本地开发可继续 fallback，真机联调需要配置后端域名和媒体域名。
- HTTPS 视频资源准备不足：第一轮只要求手动填写 HTTPS 地址，不做上传。
- 当前 mock 字段与后端字段存在差异：只保证核心课程链路兼容，非核心页面继续 fallback。
- SQLite 并发能力有限：当前一期是低并发联调场景，开启 WAL 后足够；高并发和多实例部署不进入本轮。
