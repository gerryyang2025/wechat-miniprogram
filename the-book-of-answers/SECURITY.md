# Security Policy

## Supported Scope

Security reports are welcome for the current project version on the default branch.

## How to Report

Please report security issues privately to the repository owner or maintainer.

- Do not open a public issue with exploit details.
- If GitHub private vulnerability reporting is enabled, use it.
- If no private channel is available, open a minimal issue asking for a secure contact path without sharing sensitive details.

## What to Include

Please include:

- A short description of the issue
- The affected page or feature
- Reproduction steps
- Expected impact
- Screenshots, logs, or proof of concept if safe to share privately

## Relevant Areas for This Project

Although this mini program does not use a custom backend, security reports are still useful for issues such as:

- Accidental exposure of private credentials or sensitive configuration
- Unsafe permission flows
- Unexpected remote requests or data leakage
- File export or media handling problems
- Malicious asset or script injection paths

## Response Expectations

The project aims to acknowledge valid reports promptly and to address confirmed issues in a responsible order based on severity and risk.
